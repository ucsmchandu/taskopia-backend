# Task Lifecycle in Taskopia

## Overview

Every task on Taskopia follows a structured lifecycle from creation to completion.

The lifecycle ensures that both Hosts and Allies understand what happens at each stage and when they are expected to take action.

The typical lifecycle consists of the following stages:

1. Task Creation
2. Task Publication
3. Task Discovery
4. Application Submission
5. Application Review
6. Ally Selection
7. Communication
8. Task Execution
9. Payment
10. Task Completion
11. Ratings and Reviews
12. Task Closure

---

# Stage 1: Task Creation

The lifecycle begins when a Host creates a new task.

The Host provides all the necessary information required for the work.

Typical task information includes:

- Task title
- Description
- Category
- Location
- Budget
- Date
- Time
- Estimated duration
- Special instructions

Providing complete and accurate information helps Allies understand the task before applying.

---

# Stage 2: Task Publication

After the Host submits the task, it becomes available on the Taskopia platform.

The published task becomes visible to eligible Allies.

At this stage:

- The task is searchable.
- Allies can browse it.
- Applications are accepted.

The Host can still edit or cancel the task if necessary.

---

# Stage 3: Task Discovery

Allies browse available tasks based on their interests and availability.

They may search using:

- Categories
- Keywords
- Location
- Budget
- Date

The goal is to help Allies quickly discover relevant opportunities.

---

# Stage 4: Application Submission

Once an Ally finds a suitable task, they can submit an application.

Submitting an application indicates that the Ally is interested and available for the work.

Applying does not automatically assign the task.

Multiple Allies may apply for the same task.

Similarly, an Ally may apply for multiple different tasks.

---

# Stage 5: Application Review

The Host reviews every application received.

Information available during review may include:

- Ally profile
- Ratings
- Reviews
- Completed tasks
- Experience
- Application message

The Host evaluates applicants based on the task requirements.

Taskopia does not automatically rank or assign applicants.

The final decision always belongs to the Host.

---

# Stage 6: Ally Selection

After reviewing applications, the Host selects one Ally.

Only one Ally can be accepted for a task.

Once selected:

- The Ally receives acceptance.
- Other applicants are no longer considered for that task.
- Chat becomes available.

Selection marks the transition from the application stage to the collaboration stage.

---

# Stage 7: Communication

After acceptance, the Host and Ally can communicate using Taskopia's in-platform chat.

Typical discussions include:

- Meeting location
- Arrival time
- Required tools
- Parking information
- Final instructions
- Clarifying expectations

The chat system is intended only for task-related communication.

Professional and respectful communication is expected from both users.

---

# Stage 8: Task Execution

The Ally travels to the agreed location and performs the assigned work.

Examples include:

- Cleaning
- Delivery
- Tutoring
- Furniture assembly
- Photography
- Office assistance

Task execution occurs offline.

Taskopia does not supervise or manage the work itself.

Both users are responsible for their own actions during the task.

---

# Stage 9: Payment

After successful completion, payment is handled according to the agreement between the Host and the Ally.

The payment amount should match the budget agreed upon before the task began.

Taskopia currently acts only as the platform connecting users.

Payment arrangements remain the responsibility of the participating users unless platform-supported payment systems are introduced.

---

# Stage 10: Task Completion

Once the work has been finished:

The Host verifies that the task has been completed successfully.

If satisfied, the Host marks the task as completed.

Completing the task officially ends the active work phase.

---

# Stage 11: Ratings and Reviews

After completion, both users may leave ratings and reviews.

Ratings help build trust throughout the Taskopia community.

Reviews allow users to share their experience.

Positive ratings improve user reputation.

Constructive feedback helps maintain service quality.

---

# Stage 12: Task Closure

After ratings are submitted, the task is considered closed.

Closed tasks become part of each user's history.

Task history may contribute to:

- Reputation
- Experience
- Profile strength
- Community trust

Completed tasks also help future Hosts and Allies make informed decisions.

---

# Task Statuses

Throughout its lifecycle, a task may move through several statuses.

Common task statuses include:

## Draft

The task is being created but has not yet been published.

---

## Open

The task is published and accepting applications.

---

## Under Review

The Host is reviewing submitted applications.

---

## Assigned

An Ally has been selected.

Chat becomes available.

---

## In Progress

The Ally is currently completing the task.

---

## Completed

The Host has confirmed successful completion.

---

## Cancelled

The Host cancelled the task before completion.

---

## Closed

The task has been completed and archived.

---

# Complete Workflow Example

The following example demonstrates a complete task lifecycle.

1. Ravi needs help moving furniture.
2. Ravi creates a task on Taskopia.
3. The task becomes visible.
4. Five Allies submit applications.
5. Ravi reviews all applicants.
6. Ravi accepts Priya.
7. Chat opens between Ravi and Priya.
8. They agree on the meeting time.
9. Priya completes the work.
10. Ravi pays Priya according to the agreed amount.
11. Both users leave ratings.
12. The task is marked as completed and closed.

---

# Important Notes

- A task can receive multiple applications.
- Only one Ally can be accepted.
- Chat is available only after acceptance.
- Taskopia facilitates communication but does not supervise offline work.
- Payment follows the agreement between the Host and the Ally.
- Honest ratings strengthen community trust.
- Users should report any misuse through the Report Problem feature.

---

# Summary

Every Taskopia task follows a structured lifecycle designed to create a simple, transparent, and trustworthy experience.

From task creation to completion, each stage ensures that Hosts and Allies understand their responsibilities, communicate effectively, and successfully complete short-term work while building trust within the community.
