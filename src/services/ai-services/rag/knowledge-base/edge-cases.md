# Edge Cases and Special Situations

## Overview

This document explains how Taskopia handles uncommon situations that users may encounter while using the platform.

These scenarios help users understand what they should do when unexpected problems occur.

---

# Host Accepted the Wrong Ally

## Situation

A Host accidentally accepts the wrong Ally.

## Recommended Action

If possible:

- Contact the accepted Ally through chat.
- Explain the mistake respectfully.
- Cancel the task if necessary.
- Create a new task if appropriate.

Hosts should carefully review applications before accepting.

---

# Ally Changes Their Mind

## Situation

An Ally no longer wishes to complete a task after applying.

## Before Acceptance

The Ally may simply wait or withdraw the application if the feature is available.

## After Acceptance

The Ally should immediately inform the Host through chat.

Frequent cancellations may affect reputation.

---

# Host Cancels After Acceptance

## Situation

A Host no longer needs the task after accepting an Ally.

## Recommended Action

The Host should:

- Inform the Ally immediately.
- Explain the reason.
- Cancel the task.

Hosts should avoid unnecessary cancellations.

---

# Ally Does Not Arrive

## Situation

The Ally fails to appear at the agreed time.

## Recommended Action

The Host should:

- Try contacting the Ally.
- Wait a reasonable amount of time.
- Report repeated no-show behavior if necessary.

---

# Host Does Not Arrive

## Situation

The Host is unavailable at the agreed meeting place.

## Recommended Action

The Ally should:

- Contact the Host through chat.
- Wait a reasonable amount of time.
- Report repeated no-show behavior if necessary.

---

# Payment Dispute

## Situation

The Host and Ally disagree about payment.

## Important Information

Taskopia does not currently process payments.

Users should agree on:

- Payment amount
- Payment method
- Payment timing

before starting the task.

Support may review reports but cannot enforce private payment agreements.

---

# Fake Task

## Situation

A task appears misleading or fake.

## Recommended Action

Do not apply.

Use the Report Problem feature.

Include as much information as possible.

---

# Fake Application

## Situation

A Host believes an application is fake.

## Recommended Action

Do not accept the application.

Report suspicious behavior if necessary.

---

# Inappropriate Chat Messages

## Situation

A user sends abusive or offensive messages.

## Recommended Action

Stop engaging.

Use the Report Problem feature.

Taskopia may investigate reported conversations.

---

# AI Generated Description Is Incorrect

## Situation

The AI-generated task description does not accurately describe the task.

## Recommended Action

Review the generated content carefully.

Edit it before publishing.

The Host remains responsible for the final description.

---

# AI Translation Looks Incorrect

## Situation

A translated message appears confusing.

## Recommended Action

Ask the other user for clarification.

AI translations may not always capture context perfectly.

---

# No Applications Received

## Situation

A Host receives no applications.

## Suggestions

- Improve the task description.
- Increase the offered payment.
- Choose a better category.
- Provide clearer instructions.
- Verify the location.

---

# Application Rejected

## Situation

An Ally was not selected.

## Information

Hosts choose the applicant they believe is most suitable.

Allies are encouraged to continue applying for other tasks.

---

# Internet Connection Lost

## Situation

A user loses internet connectivity while using Taskopia.

## Recommended Action

Reconnect to the internet.

Refresh the page.

Continue where possible.

---

# Chat Is Not Available

## Situation

The chat page cannot be opened.

## Possible Reasons

- No Ally has been accepted.
- The task was cancelled.
- Technical issues.
- Temporary service interruption.

---

# Forgot Account Type

## Situation

A user forgets whether they registered as a Host or Ally.

## Recommended Action

Attempt login.

Check the dashboard after signing in.

If necessary, contact support.

---

# Wrong Account Used

## Situation

A Host accidentally logs into an Ally account.

## Information

Hosts and Allies use separate accounts.

Log out and sign in using the correct account.

---

# Task Completed but Rating Missing

## Situation

A user cannot rate the other participant.

## Possible Reasons

- Task not marked as completed.
- Temporary technical issue.
- Rating period has ended.

---

# Report Submitted by Mistake

## Situation

A user accidentally submits a report.

## Recommended Action

Contact Taskopia Support and explain the mistake.

---

# Account Access Lost

## Situation

A user cannot access their account.

## Recommended Action

- Reset the password.
- Verify the email address.
- Contact support if the problem continues.

---

# Browser Issues

## Situation

Pages are not loading correctly.

## Suggestions

- Refresh the page.
- Clear browser cache.
- Update the browser.
- Try another browser.
- Check the internet connection.

---

# Uploaded Image Failed

## Situation

A profile picture fails to upload.

## Suggestions

- Check internet connectivity.
- Verify the image format.
- Reduce image size.
- Try again later.

---

# Duplicate Task

## Situation

A Host accidentally posts the same task twice.

## Recommended Action

Delete the duplicate task to avoid confusing applicants.

---

# Wrong Location Added

## Situation

A Host entered an incorrect task location.

## Recommended Action

Edit the task before it is completed.

Inform the accepted Ally if the location changes after acceptance.

---

# Frequently Asked Questions

## What if I accidentally accepted the wrong Ally?

Inform the Ally immediately and cancel the task if necessary.

---

## What if nobody applies?

Improve your task description, payment, or category and try posting again.

---

## What if AI generates incorrect information?

Always review AI-generated content before publishing.

---

## What if my application is rejected?

Continue applying for other available tasks.

Each Host makes their own selection.

---

## What if chat is unavailable?

Chat becomes available only after a Host accepts an Ally.

---

## What if someone abuses me?

Stop communicating with them and submit a report through the Report Problem feature.

---

## What if I lose access to my account?

Use the password recovery option or contact Taskopia Support.

---

# Summary

Most uncommon situations can be resolved through clear communication, responsible behavior, and Taskopia's reporting and support systems.

When users encounter unexpected issues, they should remain calm, communicate respectfully, and contact support or submit a report whenever necessary.
