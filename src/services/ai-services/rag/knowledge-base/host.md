# Hosts in Taskopia

## Who is a Host?

A Host is a registered Taskopia user who creates and publishes tasks for other users to complete.

Hosts are individuals, businesses, or organizations that need temporary assistance with specific tasks. Instead of hiring permanent employees, Hosts can quickly find suitable people to complete short-term work.

A Host has complete control over the tasks they create, the applicants they review, and the Ally they ultimately choose.

---

# Purpose of Hosts

Hosts use Taskopia to find reliable help for temporary work without going through lengthy recruitment or hiring processes.

Taskopia makes it easy for Hosts to:

- Create task listings
- Receive applications
- Compare applicants
- Select the best Ally
- Communicate securely
- Complete tasks efficiently

---

# Who Can Become a Host?

Anyone who needs temporary help can become a Host.

Examples include:

- Home owners
- Apartment residents
- Shop owners
- Small business owners
- Startups
- Event organizers
- College students
- Working professionals
- Local organizations
- Freelancers needing assistance

There are no restrictions based on profession as long as the tasks follow Taskopia's platform policies.

---

# Responsibilities of a Host

Hosts are responsible for providing accurate information about every task they create.

Their responsibilities include:

- Writing a clear task title
- Providing a detailed description
- Choosing the correct category
- Selecting the task location
- Mentioning the expected date and time
- Specifying the payment amount
- Explaining any required skills
- Reviewing applications fairly
- Communicating respectfully
- Marking tasks as completed
- Leaving honest ratings and reviews

---

# Creating a Task

Creating a task is one of the primary responsibilities of a Host.

A Host should provide enough information so that Allies fully understand the work before applying.

Every task should include:

## Task Title

A short summary describing the work.

Examples:

- Help Move Furniture
- Grocery Delivery Needed
- Garden Cleaning
- Event Photographer
- Home Tutor Required

---

## Task Description

The description should clearly explain:

- What needs to be done
- Expected outcome
- Important instructions
- Required experience (if any)
- Items provided by the Host
- Items the Ally should bring

A detailed description helps Allies decide whether the task is suitable for them.

---

## Category

Hosts should choose the category that best matches the task.

Examples include:

- Home Services
- Delivery
- Events
- Office Work
- Education
- Technical
- Personal Assistance

Choosing the correct category improves task discovery.

---

## Location

Hosts should provide the location where the task will take place.

Accurate locations help nearby Allies discover the task.

---

## Budget

Hosts should specify the payment they are willing to offer.

Providing a fair budget increases the likelihood of receiving quality applications.

---

## Date and Time

Hosts should specify:

- Preferred date
- Start time
- Estimated duration

This allows Allies to determine whether they are available.

---

# Reviewing Applications

Once a task is published, interested Allies may submit applications.

The Host can review each applicant individually.

Information available may include:

- Profile
- Ratings
- Reviews
- Completed tasks
- Experience
- Application message

Hosts should carefully evaluate applicants before making a decision.

---

# Selecting an Ally

For each task, only one Ally can be accepted.

After selecting an Ally:

- The application status changes.
- Other applicants are no longer considered.
- In-app chat becomes available.
- The Host and Ally can finalize task details.

Selection is entirely at the Host's discretion.

Taskopia does not automatically choose applicants.

---

# Chat After Acceptance

Chat becomes available only after the Host accepts an Ally.

The chat should be used for:

- Confirming meeting location
- Discussing arrival time
- Clarifying task requirements
- Answering final questions

The chat should not be used for harassment, spam, or unrelated conversations.

---

# Completing the Task

After the work is finished, the Host should:

- Verify that the task has been completed satisfactorily.
- Mark the task as completed.
- Provide payment according to the agreed terms.
- Leave an honest rating.
- Write a helpful review.

Completing these steps helps build trust within the Taskopia community.

---

# Editing a Task

Hosts can edit tasks before they are assigned or completed.

Common updates include:

- Correcting mistakes
- Updating payment
- Changing the location
- Modifying the date
- Improving the description

Significant changes should be communicated to applicants.

---

# Cancelling a Task

If circumstances change, a Host may cancel a task before completion.

Reasons may include:

- Task no longer needed
- Incorrect information
- Schedule changes
- Personal emergencies

Hosts should inform the accepted Ally whenever possible.

---

# Best Practices for Hosts

Successful Hosts generally:

- Write detailed descriptions.
- Offer fair payment.
- Respond promptly.
- Treat Allies respectfully.
- Communicate clearly.
- Complete ratings honestly.
- Close completed tasks quickly.

These practices help build a strong reputation.

---

# What Hosts Should Avoid

Hosts should never:

- Post fake tasks
- Provide misleading information
- Offer illegal work
- Harass applicants
- Discriminate unfairly
- Refuse agreed payment
- Share unnecessary personal information
- Use Taskopia for fraudulent purposes

Violating platform policies may result in warnings, suspension, or account termination.

# Frequently Asked Questions

## Can anyone become a Host?

Yes. Anyone needing temporary help can create a Host account.

---

## Can a Host create multiple tasks?

Yes. Hosts may create multiple tasks.

---

## Can a Host edit a task?

Yes. Tasks can be edited before completion.

---

## Can a Host cancel a task?

Yes. Hosts may cancel tasks if necessary.

---

## How many Allies can a Host accept?

Only one Ally can be selected for each task.

---

## Can a Host chat before accepting an Ally?

No.

Chat becomes available only after an Ally has been accepted.

---

## Does Taskopia automatically assign an Ally?

No.

The Host reviews all applications and selects the preferred Ally.

Hosts are the creators of opportunities within Taskopia.
